## 1. 배경 및 문제 정의

- 유튜브 영상을 개인 학습, 자료 보관, 오프라인 시청 목적으로 로컬에 저장할 필요가 있으나, 매번 웹 기반 서드파티 다운로더(광고, 악성 스크립트, 서비스 중단 위험)를 사용하는 것은 번거롭고 신뢰하기 어렵다.
- 재생목록/채널 단위로 여러 영상을 한 번에 받아야 하는 경우가 잦아, 단일 URL 입력을 반복하는 방식은 비효율적이다.
- 직접 만든 도구를 사용하면 다운로드 이력, 폴더 정리, 포맷 선택 등을 원하는 방식대로 제어할 수 있다.

### 사용 목적 및 제약사항

- **개인/학습용 도구**이며, 배포나 상업적 이용을 전제로 하지 않는다.
- 저작권이 있는 콘텐츠를 무단으로 다운로드·재배포하는 행위는 YouTube 서비스 약관 및 저작권법에 저촉될 수 있다. 본 도구는 **본인이 저작권을 보유했거나, 저작권자가 다운로드를 허용했거나(Creative Commons 등), 개인적 학습 목적의 사적 이용 범위 내에서만** 사용하는 것을 전제로 한다.
- DRM 우회, 유료/비공개 콘텐츠 무단 취득 등은 본 프로젝트의 범위에서 명시적으로 제외한다.

## 2. 목표 (Goals)

1. URL 하나를 입력하면 원하는 화질/포맷으로 영상을 로컬에 저장할 수 있다.
2. 재생목록 또는 채널 URL을 입력하면 포함된 영상 목록을 조회하고, 선택적으로 일괄 다운로드할 수 있다.
3. GUI에서 다운로드 진행 상황(진행률, 속도, 남은 시간)을 실시간으로 확인할 수 있다.
4. 다운로드 이력과 저장 경로를 관리할 수 있다.

### 비목표 (Non-goals)

- 웹/모바일 앱 제공 (v1은 Windows 데스크톱 GUI에 한정)
- 로그인이 필요한 비공개/멤버십 전용 영상 다운로드
- 영상 편집, 워터마크 제거 등 콘텐츠 가공 기능
- 여러 사용자를 위한 서버/멀티테넌시 운영

## 3. 타겟 사용자

- **주 사용자**: 개발자 본인 (1인 사용, 개인 PC에 설치해 로컬로 실행)
- 별도 배포/공유 계획 없음 (필요 시 추후 별도 검토)

## 4. 핵심 기능 (Feature Scope)

### 4.1 MVP (v1)

| 기능 | 설명 |
| --- | --- |
| 단일 영상 다운로드 | URL 입력 → 메타데이터(제목, 길이, 썸네일, 채널명) 미리보기 → 화질/포맷 선택 → 다운로드 |
| 재생목록 다운로드 | 재생목록 URL 입력 → 포함 영상 목록 표시(체크박스) → 선택 항목 일괄 다운로드 |
| 채널 다운로드 | 채널 URL 입력 → 업로드 영상 목록 조회(최근 N개 또는 전체) → 선택 항목 일괄 다운로드 |
| 포맷 선택 | mp4(영상), mp3/m4a(오디오만) 중 선택 |
| 화질 선택 | 사용 가능한 화질 목록(예: 2160p/1080p/720p/최고화질/최저화질) 중 선택 |
| 저장 경로 지정 | 다운로드 폴더 지정 및 기억(마지막 사용 경로 기본값) |
| 진행 상황 표시 | 개별 항목별 진행률(%), 속도, 예상 남은 시간, 전체 큐 진행 상황 |
| 다운로드 큐 관리 | 대기/진행/완료/실패 상태 표시, 실패 항목 재시도 |
| 다운로드 이력 | 완료된 다운로드 목록(제목, 날짜, 경로) 조회 |
| 에러 처리 | 비공개/삭제/지역 제한 영상 등 다운로드 불가 사유를 사용자에게 명확히 안내 |

### 4.2 확장 기능 (v2 이후, 선택)

- 자막 다운로드(자동 생성 자막 포함, srt/vtt)
- 특정 구간(시작~종료 시간) 잘라내기
- 다운로드 완료 후 파일명 규칙 커스터마이징 (예: `{채널명}_{제목}_{업로드일}`)
- 다크모드 UI
- 예약 다운로드(특정 채널 신규 영상 자동 감지 후 다운로드)

## 5. 사용자 흐름 (User Flow)

```
[앱 실행]
   │
   ▼
[URL 입력창에 영상/재생목록/채널 URL 붙여넣기] → [조회 버튼 클릭]
   │
   ▼
[URL 유형 자동 판별: 단일 영상 / 재생목록 / 채널]
   │
   ├─ 단일 영상 → 메타데이터 + 화질/포맷 옵션 표시
   │
   └─ 재생목록/채널 → 영상 목록 표시(체크박스, 기본 전체 선택)
   │
   ▼
[화질/포맷/저장 경로 설정]
   │
   ▼
[다운로드 시작] → [큐에 추가, 순차/병렬 처리]
   │
   ▼
[진행률 실시간 표시] → [완료/실패 알림]
   │
   ▼
[이력 목록에 반영, 폴더 열기 바로가기 제공]
```

## 6. 화면 구성 (초안)

- **메인 화면**: URL 입력창 + 조회 버튼, 하단에 결과(단일 영상 카드 또는 목록 테이블)
- **옵션 패널**: 화질 드롭다운, 포맷 드롭다운(영상/오디오), 저장 경로 선택
- **다운로드 큐 패널**: 항목별 상태(대기/다운로드중/완료/실패) + 진행바
- **이력 탭**: 과거 다운로드 검색/필터, 파일 위치 열기
- **설정 화면**: 기본 저장 경로, 동시 다운로드 개수, 파일명 규칙

## 7. 기술 스택 제안

| 영역 | 제안 | 비고 |
| --- | --- | --- |
| 다운로드 엔진 | [`yt-dlp`](https://github.com/yt-dlp/yt-dlp) | YouTube 정책 변경에 대응이 빠르고 재생목록/채널/자막 등 폭넓은 기능을 표준 지원. 직접 파싱 구현보다 신뢰성 높음 |
| 미디어 처리 | `ffmpeg` | 영상+오디오 병합, mp3 변환 등에 필요 (yt-dlp가 내부적으로 호출) |
| GUI 프레임워크 | Python + `PySide6`(Qt) 또는 `CustomTkinter` | PySide6: 완성도 높은 UI, 진행률/큐 관리에 적합 / CustomTkinter: 더 가볍고 빠르게 프로토타입 가능 |
| 언어 | Python 3.11+ | yt-dlp와의 연동이 자연스럽고, 포트폴리오 내 다른 스크립트와 스택 일관성 유지 |
| 패키징 | `PyInstaller` (추후) | 단일 실행파일(.exe)로 배포하고 싶을 때 사용 |
| 비동기 처리 | `QThread`(PySide6) 또는 `concurrent.futures` | 다운로드 중 UI 멈춤 방지 |

> 결정 필요: PySide6 vs CustomTkinter 중 택1은 구현 착수 전 별도로 논의.

## 8. 비기능 요구사항

- **성능**: 재생목록 조회(메타데이터만) 100개 기준 10초 이내 완료 목표
- **안정성**: 네트워크 오류/일시적 실패 시 자동 재시도(최대 3회) 후 실패 처리
- **의존성 관리**: `ffmpeg` 미설치 시 앱 실행 시점에 감지하여 설치 안내 메시지 표시
- **로깅**: 다운로드 성공/실패 로그를 로컬 파일로 기록(문제 재현 시 활용)
- **플랫폼**: Windows 10/11 우선 지원 (macOS/Linux는 비목표)

## 9. 성공 기준 (Success Metrics)

- 단일 영상/재생목록/채널 URL 각각에 대해 다운로드 성공률 95% 이상 (사용 가능한 공개 영상 기준)
- 앱 실행부터 다운로드 시작까지 3클릭 이내
- 진행 중 UI 프리징 없음(비동기 처리 정상 동작)

## 10. 리스크 및 제약사항

| 리스크 | 영향 | 대응 방안 |
| --- | --- | --- |
| YouTube 정책/구조 변경으로 다운로드 실패 | yt-dlp 자체 업데이트 실패 시 앱도 영향 | yt-dlp 버전을 주기적으로 업데이트, 실패 시 에러 메시지에 원인 명시 |
| 저작권/약관 이슈 | 법적 리스크 | 개인/학습 목적 한정 사용, 배포 계획 없음을 문서에 명시 |
| ffmpeg 미설치 환경 | 오디오 변환/병합 실패 | 최초 실행 시 설치 여부 확인 및 안내 |
| 대량 다운로드 시 IP 차단(레이트리밋) | 다운로드 실패 급증 | 동시 다운로드 수 제한, 요청 간 딜레이 옵션 제공 |
